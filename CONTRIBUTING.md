# Contributing

Thanks for your interest in NEKO.

## Ways to help
- Report bugs or request features in GitHub Issues.
- Improve docs, examples, and translations.
- Submit pull requests for fixes and improvements.

## Before you submit a PR
- Keep changes focused and scoped.
- Run `go fmt ./...` and `go test ./...` when applicable.
- Update docs if behavior or config changes.

## Workflow
1. Fork the repo and create a feature branch.
2. Make your changes with clear commit messages.
3. Open a pull request and describe the motivation and impact.

## Code of Conduct
By participating, you agree to follow `CODE_OF_CONDUCT.md`.
